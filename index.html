One line in your shell config. That's it. Breaks nothing if it fails.

user@server:~$ ssh prod-01 Welcome to Ubuntu 22.04.3 LTS ┌──────────────────────────────────────────────┐ │ Message of the Day │ │ │ │ "The best way to predict the future is to │ │ invent it." │ │ — Alan Kay│ │ tag: wisdom │ └──────────────────────────────────────────────┘ user@prod-01:~$ _
🐚
Shell (.bashrc / .zshrc)
Shows a message every time you open a terminal. Silent on failure.
source /path/to/motd/motd.sh \ 2>/dev/null || true
🔐
SSH login (Ubuntu/Debian)
Runs automatically on every SSH login via update-motd.d.
sudo cp integrate/99-motd \ /etc/update-motd.d/99-motd sudo chmod +x \ /etc/update-motd.d/99-motd
🐳
Docker / containers
Drops into /etc/profile.d/ — runs on every interactive shell.
COPY integrate/docker.sh \ /etc/profile.d/motd.sh
🌐
Remote curl (anywhere)
Fetch from a self-hosted server. 2-second timeout, never blocks.
curl -sf --max-time 2 \ https://your-server/motd \ 2>/dev/null || true
🐍
Python package
Messages bundled locally — no network required.
import motd msg = motd.random_message( tag="funny" ) print(motd.format_message(msg))
📦
npm package
Same idea, for Node.js projects.
const { randomMessage, formatMessage } = require( '@rechenmacher/motd' ); console.log( formatMessage(randomMessage()) );

Built to never break anything

Every integration path is wrapped in error handling. Missing file, unreachable server, bad JSON — all handled silently. Your shell will always open. Your SSH will always connect.
